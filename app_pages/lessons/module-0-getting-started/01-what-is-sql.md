# What is SQL?

**SQL is a language for querying data.**

Most of the time you'll use it to pull data out of a database. But SQL is so handy that it also works on data living outside databases — you can even run it against plain files.

For anyone working with data — Data Engineers, Data Analysts, Data Scientists, BI Engineers — SQL is a core skill. It's how you access databases, connect different data sources together, and describe data pipelines. If you work with data, you need SQL.

## How do you even say "SQL"?

Two ways, both correct:

- letter by letter — "ess-que-el"
- as a word — "sequel"

The name has a backstory. The ancestor of SQL was a language called DSL/Alpha, sketched out mostly on paper by Dr. Codd around 1970. A team at IBM later built a simplified version called **SQUARE**. Its second version was named **SEQUEL**, which was eventually renamed **SQL**. That language — heavily extended over the decades — is still what we use today.

You'll often see SQL described as standing for **Structured Query Language**.

## What you can do with SQL

The data usually lives in a database, and with SQL you can:

- create data and save it to the database
- retrieve, filter, and aggregate data
- transform data however you want
- update and delete data

More formally, SQL operations fall into three classes:

1. **Data statements** — manipulating data (this is what you'll use most)
2. **Schema statements** — defining data structures (tables, etc.)
3. **Transaction statements** — handling transactions (an advanced topic)

This course starts with the first class — it covers the vast majority of day-to-day work — and builds up to schema and optimization topics later.
